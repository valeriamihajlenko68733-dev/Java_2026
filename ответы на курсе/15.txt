Вывод кода:
static logic
logic (1) id= 0
logic (2) id= 1
ctor id= 100

Ответ на вопрос почему :
Статический блок – выполняется один раз при первой загрузке класса A в память (до создания любого объекта).
 Вывод: static logic

Создание объекта new A(100):
Сначала все поля объекта получают значения по умолчанию: id = 0 (для int).
Затем последовательно выполняются нестатические инициализаторы полей и нестатические логические блоки в том порядке, в котором они записаны в классе:
Первый логический блок { System.out.println("logic (1) id= " + this.id); }
На этот момент поле id ещё не инициализировано явно, поэтому его значение равно 0.
Вывод: logic (1) id= 0

Инициализация поля private int id = 1; – теперь id присваивается значение 1.
Второй логический блок { System.out.println("logic (2) id= " + id); }
Поле id уже равно 1.
Вывод: logic (2) id= 1

После выполнения всех блоков и инициализаторов вызывается конструктор public A(int id).
Внутри конструктора параметр называется так же, как поле (id), поэтому id в строке вывода относится к параметру (переданному значению 100).
 Вывод: ctor id= 100
