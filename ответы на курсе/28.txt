//Практика 1
Объснение:
Интерфейс объявлен без спецификатора доступа (package-private). Его область видимости — только внутри текущего пакета. Классы и интерфейсы из других пакетов не могут обращаться к этому интерфейсу.

//Практика 2
Объяснение:
Интерфейс объявлен как public. Область видимости — везде, где доступен сам интерфейс (т.е. любой класс из любого пакета может его использовать).

//Практика 3
Объяснение:
Для интерфейса верхнего уровня (top-level interface) модификатор protected не разрешён. Компилятор выдаст ошибку: "illegal modifier for the interface Interface; only public, abstract & strictfp are permitted". Таким образом, такой код не скомпилируется. Если бы интерфейс был вложенным (объявлен внутри класса), то protected давал бы доступ внутри пакета и подклассам внешнего класса.

//Практика 4
Объяснение:
Аналогично, для top-level интерфейса модификатор private также запрещён. Код вызовет ошибку компиляции. Для вложенного интерфейса private означал бы доступ только внутри того класса, где он объявлен.
